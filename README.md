# 企业级合规交易监控系统 (Compliance Monitoring System)

面向企业采购与供应链的制裁名单筛查、风险评分与合规审查工单全流程管理平台。

## 功能总览

| 模块 | 核心能力 |
|------|---------|
| 🛡️ **制裁筛查** | 每日自动抓取采购订单 & 供应商数据，多维度模糊匹配 OFAC / 欧盟 / 联合国等多国制裁名单 |
| 📊 **风险评分** | Jaro-Winkler + Levenshtein 算法加权评分，HS编码管制 / 国家风险 / 最终用户 / 历史风险 6 大因子 |
| 📋 **工单系统** | 高风险自动冻结 → 生成工单 → 法务审查（24h SLA，超时自动升级合规总监）→ 放行 / 拒绝 |
| 📤 **名单管理** | 制裁名单 CSV / Excel 拖拽上传，自动格式校验 & 增量更新比对库 |
| 📈 **统计报告** | 每日凌晨 02:00 自动生成报告，制裁命中率 / 审查时长 / 风险分布，支持 PDF + Excel 导出 |
| 🔍 **历史查询** | 交易编号 / 供应商 / 时间范围组合查询，流式游标处理批量导出百万级数据 |
| 📝 **审计日志** | 13 种操作分类 + 5 级严重级别，变更前后快照完整留存，满足合规审计 |
| 🔔 **实时预警** | 高风险命中 → Webhook 合规群推送（飞书 / 企业微信）+ 站内信 + 邮件多渠道投递 |
| ⚡ **高并发架构** | Bull 队列（Redis）+ MongoDB bulkWrite + 复合索引，单节点稳定支撑日 5 万+ 交易 |

## 关键配置项（`.env` 文件）

```bash
# 服务端口
PORT=3001

# 数据库（可修改为 Mongo Atlas 等云实例）
MONGODB_URI=mongodb://localhost:27017/compliance_monitor

# Redis（队列 & 缓存）
REDIS_HOST=localhost
REDIS_PORT=6379

# 风控阈值（可根据业务调优）
RISK_THRESHOLD_HIGH=80        # 高于 80 = 高风险，自动冻结+创建工单
RISK_THRESHOLD_MEDIUM=50      # 50-80 = 中风险，标记待人工抽查
REVIEW_TIME_LIMIT_HOURS=24    # 工单审查时限（小时），超时自动升级

# 合规群 Webhook（飞书 / 企业微信机器人）
WEBHOOK_COMPLIANCE_GROUP=https://open.feishu.cn/open-apis/bot/v2/hook/你的Token
```

> 环境变量修改后需 **重启后端** 生效。

## 核心模块使用指南

### 🔹 交易监控（首页 → 交易监控）
系统每天 **00:00** 自动抓取采购系统前一天交易并完成筛查。

- **手动触发同步**：点击「同步数据 → 全量同步」，等待 30-60 秒队列处理完成。
- **筛查维度**：供应商名称 / HS编码 / 原产国 / 最终用户 / 船名 / 提单号。
- **快速筛查单条**：交易列表 → 操作 →「重新筛查」。
- **批量导出**：勾选后选择「导出 Excel」或「导出 CSV」，大数量自动分 Sheet。

### 🔹 合规工单（工单管理）
- **SLA 倒计时**：工单列表「剩余时间」列，**<2h 黄色，已超时红色闪烁**。
- **审查决策**：点击工单 → 「通过」/「拒绝」按钮，拒绝时可选「同步拉黑供应商」。
- **自动升级**：每 **15 分钟** 巡检一次，已超时的普通工单自动分配合规总监，发送紧急通知。
- **工作负载**：工单列表上方显示各审查员当前工作量，用于管理员分配平衡。

### 🔹 制裁名单管理
- **上传新名单**：点击「上传名单」，拖拽 `.xlsx` 或 `.csv` 文件，系统自动校验格式并显示处理统计（新增 / 更新 / 错误行）。
- **支持字段**：`entryId, listName, entityType, primaryName, aliases, countries, hsCodes, imoNumber, remarks`。
- **关联交易回溯**：制裁条目 → 操作 →「查看关联交易」，显示所有曾命中该条目的交易。

### 🔹 统计报告（统计报告）
- **自动生成**：每日 **02:00** 自动生成前一日日报，包含 Excel + PDF 两份文件。
- **手动生成**：点击「生成报告」→ 选择日报 / 周报 / 月报 / 自定义周期。
- **核心指标**：制裁命中率、平均审查时长、风险等级分布、制裁名单命中 TOP10、审查员绩效。
- **下载**：报告列表或详情抽屉 → 下载图标（需 token，系统自动附加）。

### 🔹 通知中心
- **未读徽章**：顶部铃铛实时显示未读数，**紧急消息红色高亮**。
- **多渠道状态**：每条通知显示 站内 / 邮件 / Webhook / 短信 四种渠道投递状态（成功 / 失败 / 待发送）。
- **Webhook 测试**：通知中心 → 「测试合规群推送」，填入飞书机器人 webhook 即可验证。

| 瓶颈 | 优化方案 | 日处理能力 |

## 常见问题 (FAQ)

**Q1: 启动后端报 `MongoDB connection error`**
> A: 请检查本地 MongoDB 是否已启动，27017 端口是否可连。使用 Mongo Compass 测试连接。

**Q2: 前端页面空白 / 接口 401**
> A: 清除浏览器 localStorage 中 `token` 后重新登录。或检查前后端端口（3000 / 3001）是否被占用。

**Q3: npm install 卡在 `node-sass` 等包**
> A: 本项目不使用 sass，可换淘宝源：`npm config set registry https://registry.npmmirror.com` 后重试。

**Q4: Bull 队列警告 `Redis connection closed`**
> A: Redis 未启动。本地开发可临时关闭队列（注释 `server/config/queue.js` 初始化代码）使用同步模式调试。

**Q5: 如何接入真实采购系统 API？**
> A: 修改 `server/services/dataFetchService.js` 中 `fetchFromPOApi()` 方法，替换为真实接口调用，并配置 `.env` 中 `PO_API_URL`。

**Q6: 忘记管理员密码？**
> A: 重新执行 `npm run seed` 会重置所有用户密码为 `Admin@123`。
